Arrays and Hashes in Ruby

Sunday, May 17, 2015


For the non-technical person, an easy way to distinguish between arrays and hashes is this...

Arrays

An array contains one list of things. That list can be things that are all members of a particular category, such as the following:

colors = ["red" , "blue" , "yellow" , "green" , "brown"]

The array, colors, contains five different colors: red, blue, yellow, green, and brown. This list is ordered, which is to say that each item in the list is assigned a place. The first place is assigned to zero. The second place to 1, and so on. For example, "green" would be assigned to 3. So, whenever you want to call up an item from the list, you can simply refer to it by its place. To call up the color green, I would say the following of the array, colors:

colors[3]

This means the item in position 3 in the array, colors. Green is in that position, so green would be called up. You could then use that item in a block of code to accomplish something with it.

Hashes

Whereas an array is like a one list of items, a hash is like two lists of items. Hashes are unordered (except in later versions of Ruby), but they allow you to associate each item in the first list with an item in the second list. For example, you might want to associate one element, a person's name, with another element, his or her address. The following shows the formatting of a hash:

address_list = {"Sue Thomas" => "114 Park Ave. Oakland, CA" , "Juan Gomez" => "4450 3rd Ave. San Francisco, CA" , "Mary Johnson" => "554 Main St. Fremont, CA"}

The element on the left of the => or hash rocket, is called the key. The information on the right side of the hash rocket is called the value. While you cannot call a hash item by its numerical place (unless you assigned each value to a particular number, yourself), you can always use key to call up the value. For example, If I wanted to remember Mary Johnson's address, I would call it like this:

address_list["Mary Johnson"]

Ruby would return "554 main St. Fremont, CA" because that is the value associated with the key, "Mary Johnson".

When to Use Arrays and Hashes

As stated earlier, arrays are good containers for one list. That list will be ordered, so you can easily call up any of the items in the list according to its place in the list, which is automatically assigned a position on the list. A hash is best for situations in which you want to remember two different lists where each item in one list is associated with an item in the other list. The items in the first list will be the "keys" and items in the second list will be the "values".
