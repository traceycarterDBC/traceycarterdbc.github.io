CSS Concepts

May 10, 2015


In addition to giving you control over how elements appear on a web page, CSS allows you to control where they appear. You can, for example, tell one picture to appear below a box of text. Or you can tell one object to appear behind another object. To control how the objects are moved, we must assign each object one of four different positions: static, absolute, relative, and fixed.

STATIC

Static is what an element defaults to if you do not select any particular position. So, remember that when you have not included "position: _____" in the CSS, the object will automatically be static. An element that is static will simply fall wherever it would fall if you do not give it specific instructions on where to position itself. You do not have to do anything to render an object "static" since objects automatically default to it.

ABSOLUTE

Absolute is the position you should call an element if you want it to position itself in relation to another element. A position of "absolute" will cause an element to position itself in relation to the first of its parent elements that has a position other than static. An example would be if you wanted to have an h1 element, such as a title, center itself over a picture. Say you placed the picture exactly where you wanted it, meaning you did not simply let it fall where it wanted - in other words, you did not leave its position as static by default. To center a title above the picture, you should make that title element a child of the picture. Then, give the title element a position of "absolute." Then you can center the title element. This will make the title element a child of the picture, which we said was not "static." So, the title element will position itself relative to its parent, the picture, an image element. RELATIVE

Relative is the position you would give to an element if you want to tell it where to move relative to where it would have been if it had been the default static position. For example, if you gave a

element a position of "relative," you could tell it to be 20 pixels down from where it would have been if it had been static.

FIXED

Fixed is the last type of position in CSS. An element that is "fixed" is anchored to the browser window. When you scroll up and down the page, other elements (that are not set to "position: fixed") will move, but a fixed element will not.
