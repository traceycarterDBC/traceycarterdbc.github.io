JavaScript: Constructor Function vs. Literal Notation

  • Constructor Function --> Function Declaration
  • Literal Notation --> Function Expression

Constructor Function

  • Defines a named function variable without requiring a variable assignment
  • Function declarations occur as a stand-alone construct and cannot be nested within non-function blocks.
  • Function declarations start with "function" (as variable declarations start with "var")
  • Also known as Function Declaration

Example of Constructor Function

function bar() {}

return 3;

}

Literal Notation

  • Defines a function as a part of a larger expression syntax (typically a variable assignment)
  • Functions defined via function expressions can be named or anonymous, but must NOT start with "function".
  • Also known as Function Expression

Examples of Function Expression

Anonymous Function Expression

var a = function() {

return 3;

}

Named function expression

var a = function bar() {

return 3;

}
